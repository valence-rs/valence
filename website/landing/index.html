A Rust framework for building Minecraft servers.

Built on top of Bevy ECS, Valence is an effort to create a Minecraft compatible server completely from scratch in Rust. You can think of Valence as a game engine for Minecraft servers. 

use valence::prelude::*;

pub fn main() {
    tracing_subscriber::fmt().init();

    App::new()
        .add_plugins(DefaultPlugins)
        .add_startup_system(setup)
        .add_system(init_clients)
        .add_system(despawn_disconnected_clients)
        .run();
}

Goals

Complete

Abstractions for the full breadth of the Minecraft protocol.

Flexible

Easily extend Valence from within user code. Direct access to the Minecraft protocol is provided.

Modular

Pick and choose the components you need.

Intuitive

An API that is easy to use and difficult to misuse. Extensive documentation and examples are important.

Efficient

Optimal use of system resources with multiple CPU cores in mind. Valence uses very little memory and can support thousands of players at the same time without lag (assuming you have the bandwidth).

Up To Date

Targets the most recent stable version of Minecraft. Support for multiple versions at once is not planned. However, you can use a proxy with ViaBackwards to achieve backwards compatibility with older clients.

Examples

Cow Sphere

Leverage Bevy's entity control system to make a fibonacci spiral of cows.

Anvil Loading

Use valence_anvil to load a regular Minecraft world file into your custom server.

Combat

Create your own combat mechanics with arbitrary structured client state.
